## Plan: 命令行版需求细化 V1

基于你确认的方向，我会把 `需求.md` 重构为“先 CLI、后 GUI”的两阶段需求文档：第一阶段聚焦命令行可用闭环（搜索、基础元数据、视频/音频下载），第二阶段预留 GUI 演进接口。文档将明确功能边界、输入输出、错误处理和验收标准，确保后续可直接拆任务开发。

### Steps
1. 重写 `需求.md` 的目标与范围，明确 `CLI MVP` 优先级。  
2. 定义搜索流程：关键词输入、结果列表字段、下一步选择交互。  
3. 定义基础元数据能力：链接解析与标题/简介/播放/弹幕输出。  
4. 定义下载能力：视频与音频下载参数、保存路径、格式与清晰度。  
5. 补充非功能与验收：失败重试、日志提示、扩展到 `GUI Phase` 条件。  

### Further Considerations
1. CLI 交互风格确认：子命令模式 / 交互问答模式 / 二者兼容。  
2. 下载“无能力限制”是否包含：批量下载、合集/分P、断点续传。  
3. 基础元数据最终字段集确认：仅4项 / 增加时长与UP主 / 增加发布时间。
